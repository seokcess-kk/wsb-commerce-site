# WSB 커머스 — 고객 여정 갭 분석 & 우선순위 백로그

> 작성일: 2026-06-09 · 범위: 고객 여정 전체(홈~재방문), 어드민·운영 제외 · 근거: 확립된 이커머스/국내 종합몰 모범사례 + 실제 코드 확인

## 0. 목적과 방법론

현재 WSB 종합몰(건기식 D2C)은 "보고 → 담고 → 결제 → 받고 → 재방문" 핵심 흐름의 MVP가 구현된 상태다. 이 문서는 **일반적인 제품 판매 사이트가 갖춰야 할 표준 기능·화면·UI/UX 구성요소**를 기준으로 현재 구현을 대조해, 무엇이 있고 무엇이 빠졌는지 판정하고, 빠진 것에 **임팩트×노력 기반 우선순위(P0/P1/P2)**를 부여한다.

### 현재 상태 판정 기준
- ✅ **있음** — 코드에 구현되어 동작
- 🟡 **부분** — 일부만 있거나, 어드민에만 있고 고객 화면엔 없는 등 불완전
- ❌ **없음** — 미구현

판정 근거는 모두 실제 코드(`src/app`, `src/components`, `src/db/schema`)를 직접 확인해 기록했다. 추정하지 않는다.

### 임팩트×노력 루브릭
- **임팩트** (高/中/低): 매출·전환·리텐션에 미치는 영향. 건기식 D2C 특성(반복구매·신뢰·규제)을 가중.
- **노력** (S/M/L/XL): 구현 규모. 기존 스키마·컬럼·패턴 재사용 가능성 반영.
- **우선순위 매핑**
  - **P0** = 高임팩트 × (S/M노력) — 즉시 후보
  - **P1** = 高임팩트 × L노력 **또는** 中임팩트 × (S/M노력)
  - **P2** = 그 외(低임팩트 또는 고비용 대비 효과 낮음)

## Part 2. 기능 도메인별 갭 심층 분석

**A. 상품 탐색·발견** — 카테고리 필터는 있으나 정렬·속성 필터·페이지네이션이 없다. 상품 수가 늘면 "원하는 걸 못 찾는" 이탈이 발생한다. 검색은 동작하나 자동완성/추천검색어가 없어 입력 부담이 크다.

**B. 상품상세·전환 (가장 약한 고리)** — PDP가 실제 상품 이미지를 한 장도 보여주지 않는다(컬러 placeholder). 건기식은 패키지·성분 이미지가 구매 결정의 핵심인데 시각 정보가 0이다. 수량 선택·바로구매·리뷰·연관상품도 없어 전환·객단가를 끌어올릴 장치가 비어 있다.

**C. 장바구니·체크아웃** — 결제창 직전에 주문상품 목록을 보여주지 않고(총액만), 주소를 단일 텍스트로 받아(우편번호 검색 없음) 배송 오류·불안을 유발한다. 한국 커머스 표준 대비 신뢰 격차가 있다.

**D. 회원·인증** — 로그인/소셜은 양호하나 비밀번호 재설정·약관 동의 분리·비회원 주문조회(FAQ에서 약속했으나 미구현)가 빠져 법적·신뢰 리스크가 있다.

**E. 리텐션 (재방문 엔진 — 사실상 공백)** — 리뷰·위시리스트·재구매·적립이 전무하다. "재방문" 흐름을 표방하지만 돌아올 이유를 만드는 장치가 없어 건기식 D2C의 반복구매 LTV를 살리지 못한다.

**F. 프로모션** — 쿠폰·적립·기획전이 없다. 매출 레버가 없어 신규 유입을 전환·재구매로 묶지 못한다.

**G. CS·신뢰** — FAQ·정책은 있으나 고객이 직접 취소/교환/반품을 요청할 경로가 없고(전자상거래법상 청약철회), 1:1 문의 폼·공지도 없다.

## 권고

- **즉시 착수:** P0 4건은 모두 기존 데이터·컬럼을 재사용하는 "퀵윈"이며, 전환율을 단기간에 끌어올린다. 하나의 구현 플랜으로 묶어 진행 가능.

- **다음 단계:** P0 묶음에 대한 구현 플랜 작성(`writing-plans`).
