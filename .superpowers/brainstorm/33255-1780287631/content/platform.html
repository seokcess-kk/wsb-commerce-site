구축 방식을 어디로 잡을까요?

WSB 종합몰(NUTROGIN + 기존 건기식)을 어떤 기반으로 만들지 — 이후 일정·비용·기능 자유도를 모두 가르는 결정입니다. 마음이 가는 쪽을 눌러주세요.

A

커스텀 개발 — Next.js + Vercel 추천

프론트·백엔드를 직접 짭니다. 브랜드 표현·정기구독·번들·UX를 100% 자유 설계.

강점

  • WSB/NUTROGIN 브랜드 가이드 픽셀 단위 구현
  • 정기구독·세트·진단형 추천 등 차별화 기능 자유
  • 데이터·소스코드 완전 소유, 확장 무제한

부담

  • 결제(PG)·배송·정산·세금계산서·CS를 직접 연동
  • 개발 기간·비용 가장 큼 (통상 14~18주+)
  • 운영/보안/장애대응 책임이 우리에게
B

국내 커머스 솔루션 — 카페24 / 아임웹 등

검증된 쇼핑몰 솔루션 위에 디자인만 얹습니다. 건기식 운영 사례 다수.

강점

  • 결제·배송·정산·세금계산서·CS·회원 전부 내장
  • 최단 오픈, 운영 인력 진입장벽 낮음
  • 건기식 PG 심사·통신판매 운영 노하우 풍부

부담

  • 디자인·인터랙션·커스텀 기능에 솔루션 제약
  • 브랜드 가이드 100% 구현은 어려움
  • 월 사용료·트래픽/기능별 종속
C

하이브리드(헤드리스) — 커스텀 프론트 + 검증된 커머스 백엔드

프론트는 Next.js로 자유롭게, 주문·결제·정산은 Shopify 등 검증된 엔진에 위임.

강점

  • 브랜드 프론트 자유 + 결제/정산은 안정적 엔진
  • 커스텀 A의 운영 리스크를 상당 부분 흡수

부담

  • 구성 복잡도·연동 비용 가장 높음
  • 국내 결제/세금계산서 대응에 추가 작업
